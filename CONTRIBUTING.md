# Contributing to the project

## New feature
1. For new feature requests, first need to raise issue for requesting new feature
2. Maintainers will review the feature request and based on the issue activity will assign the issue to a user
3. User is supposed to work on side branch named `feature/<featureName>`
4. Once the coding part is done, user should raise pr and wait till maintainer reviews the code and in pr description issue number should be linked and in issue description pr link should be added
5. Once approved, reviewer will merge the code and closes the issue.


## performance issues
1. For performance issues, first need to raise issue for requesting performance issue fixes
2. Maintainers will review the issue request and based on the issue activity will assign the issue to a user
3. User is supposed to work on side branch named `performance/<perfornamceIssueName>`
4. Once the coding part is done, user should raise pr and wait till maintainer reviews the code and in pr description issue number should be linked and in issue description pr link should be added
5. Once approved, reviewer will merge the code and closes the issue.
